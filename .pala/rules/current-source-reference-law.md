# Current Source Reference Law

Pala OS standards must stay current.

## Current-source rule

If a claim depends on changing tools, docs, model behavior, MCP syntax, GitHub readiness, package publishing, or security best practices, verify current sources before turning it into a rule.

## Minimum evidence

For a world-class claim, store at least two fresh references when possible:

```txt
docs/evidence/reference-radar.md
.pala/state/reference-radar-state.json
.pala/ledger/reference-refresh.jsonl
```

## No-copy rule

References are used for learning and benchmarking only.
Do not copy competitor code, branding, README wording, UI copy, package names, or screenshots.
