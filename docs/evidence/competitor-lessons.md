# Competitor / Reference Lessons

Checked on 2026-06-04.

| Reference | Lesson | Pala Boundary |
|---|---|---|
| OpenHands | Mature coding-agent systems expose observable execution surfaces. | Pala stays above the coding agent as verifier/control layer. |
| OpenCode | Terminal UX benefits from sessions, persistence, and file-change visibility. | Pala uses local SQLite, ledger, and evidence without copying agent behavior. |
| WrongStack | Autonomy still needs permission visibility. | Pala keeps approval and dry-run gates explicit. |
| Backstage | Catalogs and metadata create a useful developer control tower. | Pala dashboard reads local truth sources only. |
| OpenHands Local GUI | A local REST API and SPA can make agent operation visible. | Pala keeps its UI a mirror of DB/state/evidence instead of an execution authority. |
| Langfuse / Helicone | Usage and cost observability need source and confidence. | Pala separates exact, estimated, and unknown token data. |
| OpenSSF Scorecard | Automated security checks are useful signals, not absolute truth. | Pala reports readiness without unsupported guarantees. |

No source code, branding, README wording, UI copy, package name, or screenshot is copied.
