# Pala OS Technical Story for Presentation

## Slide story

1. Problem: AI coding agents say done without enough proof.
2. Positioning: Pala OS is not the agent; it is the local control/evidence layer.
3. ZIP: a ready-to-drop session pack that gives the agent rules, order, examples, and gates.
4. Kernel: rules, state, ledger, evidence.
5. Database: SQLite gives local app memory.
6. Decision engine: memory, source, risk, token, dry-run, approval, execution, evidence.
7. Dashboard: reads truth from DB/state/ledger/evidence.
8. CLI: before, during, and after coding commands.
9. MCP: safe dry-run repair, no real config write without approval.
10. Token economy: exact vs estimated, confidence, no fake cost claims.
11. Reference radar: learn from strong projects without copying.
12. GitHub readiness: public repo quality, security, contribution, honest install status.
13. First 10 minutes: prove value quickly.
14. Final line: Agent does the work. Pala OS verifies the work.

## Backend / frontend explanation

Backend writes truth. Frontend shows truth. Worker refreshes truth. Decision engine gates truth. Evidence proves truth.
