# Pala OS Positioning

Pala OS is not a coding agent.

Pala OS is a local control tower and operating layer for AI/vibe coding agents.

## One-line public positioning

Pala OS turns AI coding agents into evidence-driven, memory-aware, cost-aware, dashboard-visible operators.

## Short tagline

Agent does the work. Pala OS verifies the work.

## What Pala OS does

- Tracks repo memory in `.pala/rules`, `.pala/state`, `.pala/ledger`, `.pala/evidence`, and local SQLite.
- Requires evidence before "done".
- Checks README/docs/panel/package/state drift.
- Shows local control state through dashboard routes and `.pala/state/dashboard-state.json`.
- Manages MCP setup with dry-run, repair, backup, and preservation principles.
- Tracks token economy with exact known usage separated from estimates.
- Keeps benchmark/reference radar current before standards.
- Prepares public GitHub readiness without fake publication claims.

## What Pala OS does not claim

- It is not a replacement for Claude Code, Cursor, Codex, OpenCode, or OpenHands.
- It is not a general autonomous coding agent.
- It is not unlimited autonomy.
- It does not push, publish, delete, or modify real user configs without approval.
- It does not claim exact token/cost data unless measured.

## Internal analogy

A fair internal analogy is:

> Pala OS helps a vibe coder run AI agents with the discipline of a high-value senior operator: evidence, memory, cost tracking, drift checks, and dashboard control.

Use this analogy carefully. Do not turn it into an unsupported technical claim.
